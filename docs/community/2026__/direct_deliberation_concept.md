**記録日**: 2026-02-25

## 概念の整理

### 現在の制度：間接合議制

```
住民
  ↓（班ごとの輪番で）
班長（各班の代表として役員会に参加）
  ↓
役員会で合議（三役3名 + 班長6名 = 9名）
  ↓
決定
```

住民の意思は、**班長を介して**役員会に届く。  
班長は輪番制であり、必ずしも「住民の意見を集約する代表」として機能しているわけではない。  
班長が役員会で何を話し合ったか、住民に還元される保証もない。

この構造を「間接合議制」と呼ぶ。

---

### 対比：直接合議制

住民が班長を介さずに、意思決定の場に**直接参加**する仕組み。

現在の自治会制度のなかで、実質的な「直接合議」の実例はすでに存在している：

| 場・手段 | 性格 |
|---|---|
| **総会** | 会員全員が参加できる最高議決機関（ただし現状は新旧役員18名＋書面回答） |
| **観音台カフェ** | 役員・会員が自由に参加する対話の場（非公式だが実質的な直接合議） |
| **アンケート** | 全会員が直接意見を表明できる手段 |
| **個別相談** | 会員が事務局に直接連絡・相談できる窓口 |

これらは、班長という中間層を経ずに、住民の意見が意思決定に届く経路である。

---

## 三層モデルの今昔の対比との接続

年報案 v2 のセクション2「期待の三層モデル」は、「昔」と「今」の対比として、住民の**期待の多様化**を描いている。この視点は、その対比と同じ社会変化の裏面である。

| | 昔 | 今 |
|---|---|---|
| **住民の期待** | 均質（三層すべてを自治会に依存） | 多様化（特に第三層は個人が自充足） |
| **班長の代表性** | 機能しやすい（班の意見がおおむね一致） | 機能しづらい（各世帯の期待が異なる） |
| **間接合議の有効性** | 高い | 低下している |
| **直接参加の必要性** | 低い | 高まっている |

サービス構造（何を提供するか）と意思決定構造（どう決めるか）の両方が、**同じ社会変化によって同時に陳腐化しつつある**。これが「直接合議制への移行」という視点の根底にある認識。

言い換えれば：住民の期待が均質だった時代には、班長が「班の代表」として役員会に意見を届ける仕組みは合理的だった。しかし期待が多様化した今、班長一人が班全員を代表することは構造的に無理になっている。

---

## この視点が意味すること

### 1. 「班長の負担」問題の根本的な見直し

現在の問題（班長のなり手不足）は、「班長を見つける」という発想で対処されてきた。  
しかし「直接合議制への移行」という視点に立てば、**班長が担っている機能のうち、住民↔役員会の橋渡し部分を、別の手段で代替できないか**という問いが生まれる。

- 班長の役割の一部（情報伝達、意見集約）を、観音台カフェ・アンケート・メーリングリスト等で代替
- 班長は主に「回覧・集金」等の物理的業務の担当に絞る
- 意思決定への住民参加は、直接参加型の場を整備することで確保

### 2. 意思決定の質の向上

班長を介した間接合議では、**班長の理解力・関心・表現力**に情報の質が依存する。  
直接合議制では、関心を持つ住民が直接参加するため、議論の深さと多様性が向上する可能性がある。

「本音が言える信頼関係を築きながら、時間をかけて醸成していく」（意思決定プロセスの改善施策）は、この方向の第一歩とも言える。

### 3. 現行4施策との接続

| 施策 | 直接合議制との関係 |
|---|---|
| わたしがやります制度 | 班の枠を超えた個人の直接参加の慣例化 |
| 賛助会員（アンケート先行型） | 全会員への直接意見聴取を制度化 |
| なかなおり懇談会 | 対立当事者の直接対話（班長という媒介を外す） |
| 意思決定プロセスの改善 | 直接合議の場（カフェ・談話会等）の体系化 |

## 現時点での位置づけ

- 「直接合議制への移行」は現時点では**制度提案ではなく、方向性の認識**である
- 会則の「役員会が決定機関」という構造は変えない前提で、その補完として直接参加の場を拡充する
- 長期的には、会則改正を含む制度設計の議論につながりうる

## 技術的な側面：直接参加を支えるツール

直接合議制への移行を実質的に支えるのは、期待の多様化という社会的背景だけでなく、**ツールの進化**でもある。

### 可能性

| ツール | 現状 | 可能性 |
|---|---|---|
| 紙のアンケート | 実施済み（冬・夏） | 設問設計の精緻化・結果の可視化 |
| オンラインアンケート | 未実施 | 集計の省力化・回答率向上の可能性 |
| オンライン投票 | 未実施 | 書面回答方式の総会を部分的に代替できる可能性 |
| メーリングリスト・グループ | 未実施 | 班長を介さない情報共有・意見交換 |
| AI活用（年報・記録） | 今年度から試行 | 記録の蓄積・引き継ぎの自動化 |

### IT格差への配慮（必須）

ただし、技術の導入は**IT格差の問題を必ず伴う**。

- 現在の紙文書による運営は、デジタル環境を持たない住民も等しく参加できることが前提
- 角倉会長がメールを使わない事例（→ `2026_chairman_communications.md`）に象徴されるように、デジタル化を進めると**関与できなくなる層が生まれる**
- 「技術を使える人だけが参加し、意見を届けやすくなる」構造は、直接合議制の精神に反する

### 方向性：ハイブリッド型

- **紙とデジタルの並立**：デジタルが使える人はオンラインで、使えない人は従来通り紙・対面で参加できる並立運営
- **デジタル化は補完であって代替ではない**：紙の回覧・投函・対面を廃止するのではなく、選択肢を増やす方向に限定する
- **IT格差の把握から始める**：住民のデジタル環境の実態（メール利用率・スマートフォン所持率等）をアンケートで確認してから設計する

この慎重さは、賛助会員制度の「アンケート先行型」（→ `2026_nakayama_consultation_20260225.md`）と同じ思想である：**「しっかりと結果が予測できる状況を作って進める」**。

### IT格差への配慮と前進のバランス（重要）

ただし、配慮には非対称性がある。

**配慮が「中心」になると、前進そのものが成り立たなくなる。**

- 角倉会長のような「メールを使わない人」への配慮は必要である
- しかし、その配慮を理由に変化を止めると、「先を行く」方向そのものが消える
- 技術に不慣れな人が反発するのは、変化一般への自然な反応であり、変化しないことの根拠にはならない

**整理すると：**

| 配慮の内容 | 妥当か |
|---|---|
| 紙の手段を残す・廃止しない | ✅ 必要 |
| デジタルでも参加できる選択肢を提供する | ✅ 推進すべき |
| 反発を避けるために導入自体を見送る | ❌ 配慮の過剰適用 |
| 「理解してもらってから進める」を理由に無期限延期 | ❌ 前進の否定 |

現実的な進め方は、**「使いたい人が使えるようにする」ことを先行させ、「使わない人を排除しない」ことを保証する**という順序である。この順序を逆にすると、常に最も保守的な人のペースに合わせることになり、制度が時代に取り残されていく。

角倉会長への対応（連絡書投函・ご返信不要）も、同じ構造である：配慮はするが、それを中心に置かない。

## コンビニ払いとの接続（直接合議の実践例）

コンビニ払い（→ `2026_payment_method_reform.md`）を検討したときにも、この視点は既に存在していた。

### 払込用紙を「参加のチャネル」として使う

コンビニの払込用紙は、全会員が必ず手にする物理的な接点である。支払い行為そのものが、班長を介さない**全会員への直接リーチ**になる。

これを活用すると：

| 活用例 | 内容 |
|---|---|
| **賛否の確認** | 同じ金額でも別の払込用紙を用意し、選択させる（例：A案・B案の2種類） |
| **募金の選択制** | 赤十字社への募金を「含む／含まない」の2択で払込用紙を分ける |
| **簡易アンケート** | 払込用紙の裏面や別紙に設問を添付し、支払い時に一緒に提出 |
| **その他の情報収集** | 金額だけでなく、選択肢や記述欄を組み合わせた多様な形式が可能 |

### なぜこれが「直接合議」なのか

- 班長が集約・代表するのではなく、**各世帯が直接意思を示す**
- コンビニ払いはデジタルを必要とせず、IT格差の問題が生じない
- 払込という「用事のついで」に参加できるため、参加コストが低い
- 集計結果が定量的に残る（アンケートより回答率が上がる可能性）

### コンビニ払いの本質的な優位性

コンビニ払いは、IT格差がほとんど存在しない方法である。

- スマートフォン・メール・PCを必要としない
- 高齢者・デジタル非利用者も同じ手順で利用できる
- 「IT格差への配慮」と「情報収集の効率化」を同時に満たす、数少ない手段

この点で、オンラインアンケートや電子投票とは本質的に異なる。「デジタル化だが格差を生まない」手段として、優先度が高い。

### サービス提供会社への提案という視点

払込用紙を情報収集チャネルとして活用する機能は、自治会側が独自に設計するだけでなく、**コンビニ払いサービスの提供会社に提案・要望する**という方向もある。

- 自治会向けコンビニ払いサービスを提供する会社（例：地銀系・決済代行系）はすでに存在する
- 「払込用紙のバリエーション設定」「回答欄の追加」「集計レポートの提供」などの機能が、すでに一部のサービスで提供されている可能性がある
- 存在しない場合、要望として提案することで、他の自治会にも有益な機能追加につながりうる

これは単なる利用者の要望にとどまらず、**「自治会のDXを支えるサービス設計への貢献」**という位置づけになりうる。年報案の「先を行く」という姿勢とも一致する。

現在の意見収集手段：
- 役員会（班長のみ参加）→ 間接
- 総会（書面回答）→ 直接だが年1回・議案に限定
- アンケート（紙）→ 直接だが配布・回収に班長の手間がかかる
- 観音台カフェ → 直接だが参加者が限定的

払込用紙の活用は、**「全会員が必ず参加する行為」に意見収集を組み込む**点で、他の手段より構造的に優れている。

### 課題

- 払込用紙の設計・印刷コストの増加
- コンビニ払い自体がまだ実現していない（→ 先にコンビニ払いの導入が必要）
- 選択肢の設計が不適切だと、誘導や混乱を生む可能性

→ コンビニ払い導入の意義として、「支払いの簡便化」に加えて「直接参加チャネルの確保」を明示することで、導入提案の説得力が増す。

## 年報案への反映（2026-02-25）

`annual_report_part1_draft_v2.md` のセクション4「健康な構造のビジョン」末尾に段落を追加。

**要旨（年報案の表現）**：
- 現状：住民の意見が「班長→役員会」という経路をたどる（間接）
- 班長が全員の意見を集約・代表するのは実際には困難 → 引き受け不足の一因
- 観音台カフェ・アンケートはすでに直接参加の場として機能している
- 班長の役割を「橋渡し」から「地域の実務担当」に絞ることで負担軽減
- 専門用語「間接合議制」は使わず、一般読者向けの平易な表現を採用
